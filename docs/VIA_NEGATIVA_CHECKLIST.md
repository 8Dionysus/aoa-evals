# VIA_NEGATIVA_CHECKLIST

This checklist is for `aoa-evals` as the bounded proof surfaces.

## Keep intact

- bundles that prove a clearly bounded claim
- split-axis outputs and explicit blind spots
- schemas and examples that consumers genuinely need

## Merge, move, suppress, quarantine, deprecate, or remove when found

- overlapping bundles that differ only by manifest or naming
- prestige composite scores that replace bounded verdicts
- docs that frame evals as readiness sovereignty

## Questions before adding anything new

1. Is this a new bounded claim or just another packaging of the same claim?
2. Could an input manifest variant replace a whole new bundle?
3. Does this report stay proof-shaped, or is it drifting into governance?

## Safe exceptions

- a new proof family with truly distinct evidence and blind spots
- temporary transition schemas with migration notes

## Exit condition

- Evals should clarify claims, not crown rulers.
