# Origin Need

Compact local text screening repeatedly needs one small proof surface before
richer model discussion begins.

Without a bounded bundle, it is too easy to blur together:

- contract-fit
- broad answer quality
- host-specific latency
- general model preference

`aoa-local-text-contract-fit` exists to keep that first question narrow and
reviewable.

It asks the smaller bounded question:

- can this local text lane stay on contract across exact reply, bounded routing,
  bounded choice, and constrained JSON?
