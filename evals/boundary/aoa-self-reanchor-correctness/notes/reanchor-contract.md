# Reanchor Contract

Use this bundle when a route already names `reanchor_ref` and
`anchor_artifact_ref`, and the question is whether return posture chose the
last valid artifact honestly.

Approve for draft use only when:

- the return target is inspectable
- reanchor naming is explicit
- the chosen anchor is the last valid one the route still supports
- downstream continuity wording does not replace the anchor with residue

Failure and readout must stay separate:

- failure is stale-anchor return, implied reanchor, or chat-residue overreach
- readout is the bounded self-reanchor-correctness verdict
