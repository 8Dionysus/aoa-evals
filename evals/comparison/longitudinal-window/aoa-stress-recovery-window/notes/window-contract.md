# Window Contract

Use this bundle only when the window sequence is:

- ordered
- named
- comparable on the same bounded stress recovery family
- backed by source-owned owner receipts and one public report artifact for the window

Each window should preserve:

- the same owner surface or tightly related surface family
- the same named stressor family
- the same bounded evidence ordering from owner receipts outward
- the same anchor workflow surface, namely `aoa-antifragility-posture`, for the repeated-window read

Each window should disclose:

- context drift that changes comparability
- whether owner evidence thinned enough to force caution
- whether route hints or memo context were secondary rather than primary
- why the honest read is `no clear directional movement`, `mixed`, or another bounded posture when evidence is not clean

Treat style-only or report-only movement as:

- `no clear directional movement`
- or a more cautious `mixed` read

unless the owner-first recovery evidence itself also moved.
