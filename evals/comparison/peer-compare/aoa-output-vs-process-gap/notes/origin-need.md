# Origin Need

The repository now has public workflow starters and is about to gain a public artifact starter,
but it still lacks a bridge surface between them.

That makes it possible to ask:
- is the workflow strong?
- is the artifact strong?

It does **not** yet make it possible to ask the bounded bridge question:
- is polished output outrunning process discipline,
  is process discipline outrunning artifact strength,
  or are the two broadly aligned on the same cases?

`aoa-output-vs-process-gap` exists to separate that comparative question
from nearby surfaces such as:
- standalone artifact review
- standalone workflow review
- outcome-versus-path separation

This bundle should make artifact/process divergence visible
without pretending to explain every root cause behind the gap.
