# Portable Review

This bundle is portable because it does not depend on one repository, one exact
ring name, or one transcript format.

To preserve portability, downstream replacements must keep:

- a named ring or skill family request
- explicit applicability-map evidence
- a visible execution-versus-closeout distinction
- a visible harvest check or explicit no-harvest decision

This bundle should remain portable-bounded rather than broad:

- it can travel across repositories that use ring or family language
- it should not become a general reasoning or planning leaderboard
