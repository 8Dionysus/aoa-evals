# Bounded Promotion Review

This note is the `support_note` for moving `aoa-trace-outcome-separation` from `draft` to `bounded`.

Outcome: approve for bounded promotion

Approve for bounded promotion when:
- the bundle keeps outcome and path notes separate before any combined reading
- polished outcome does not erase weak path evidence
- a disciplined path does not overstate an incomplete outcome
- the public readout keeps this split distinct from artifact-versus-process comparison

Defer bounded promotion when:
- the combined reading appears without separate outcome and path notes
- the report launders visible divergence into a clean workflow pass
- one exact trace is implied as the only disciplined path
- the public wording collapses this split into a generic workflow score

Failure vs readout:
- failure is the mismatch between the separate outcome/path evidence and the combined claim
- readout is the public summary of that mismatch
- a clean readout cannot repair a flattened split
- a clumsy readout does not by itself invalidate a supported separation
