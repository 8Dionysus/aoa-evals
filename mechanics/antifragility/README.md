## Entry Route

Start with this README for role and owned operation. Then read [DIRECTION.md](DIRECTION.md) for current operating direction, [PARTS.md](PARTS.md) for active parts, and [PROVENANCE.md](PROVENANCE.md) only for legacy or former placement.

## Role

`mechanics/antifragility/` routes bounded proof work for stress posture,
recovery windows, and repair evidence on the eval side.

It receives stress or repair pressure after source owners and candidate
evidence routes are visible, then hands bounded readings to source bundles,
reports, comparison routes, audit routes, or owner handoffs.

## Owned Operation

`mechanics/antifragility/` owns the eval-side antifragility proof operation:

`stress or repair pressure -> bounded antifragility proof question -> owner-first evidence -> bundle-local review -> bounded report or owner handoff`

This package is AoA-aligned. It keeps the parent name `antifragility` because
the operation materializes the center antifragility mechanic on the proof side.

## Parts

See [PARTS.md](PARTS.md).

The active parts are `posture-review`, `stress-recovery-window`, and
`repair-proof`. Diagnosis-cause discipline is not an antifragility part; it
routes through `mechanics/growth-cycle/parts/diagnosis-gate/` while bounded
repair proof remains here.

## Inputs

- source-owned stressor receipts and owner-local artifacts;
- degraded continuation, safe-stop, re-entry, quarantine, or regrounding
  evidence;
- ordered stress recovery windows when longitudinal movement is claimed;
- repair or reanchor artifact trails when bounded repair proof is claimed;
- selected runtime or trace evidence from `mechanics/audit/` as candidate-only
  sidecar input.

## Outputs

- bounded antifragility posture reports;
- split-axis stress and recovery readouts;
- bounded repair-proof notes;
- owner handoff requests or adjacent bundle routes;
- no one-score health metric, deletion theater, runtime recovery authority, or
  global resilience claim.

## Stronger Owner Split

`Agents-of-Abyss` owns antifragility doctrine, via negativa, anti-authority
posture, and owner-request vocabulary. Owner repositories own local incidents,
runtime behavior, deletion, cleanup, repair execution, and source receipts.
`aoa-memo` owns retained lessons. `aoa-stats` owns derived fragility windows
without health-score authority. `aoa-playbooks` owns recurring degraded-mode
choreography.

`aoa-evals` owns bounded proof wording, verdict logic, report interpretation,
repair proof, regression evidence, and the review boundary around selected
evidence.

## Stop-Lines

This package supports bounded eval-side proof only. Keep these claims outside
this package:

- global resilience or federation health;
- live runtime self-healing;
- deletion, cleanup, or owner-local repair authority;
- one-score antifragility;
- proof that route hints, stats, memo context, or generated readers outrank
  owner receipts;
- proof that diagnosis-cause discipline or growth-cycle movement is solved.
