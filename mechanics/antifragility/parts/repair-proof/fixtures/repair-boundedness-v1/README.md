# Repair Boundedness V1

This shared family anchors `aoa-repair-boundedness`.

Use it when the public question is:
- did a repair or reanchor move stay bounded, preserve owner boundaries, and leave a reviewable trail instead of smearing scope across layers?

Canonical case archetypes:
- one honest bounded repair with a clear target seam
- one widening-shaped repair that hides scope inflation behind successful-looking prose
- one reanchor move that narrows ambiguity without widening authority
- one trail-gap case where the move may help locally but does not leave enough reviewable evidence

Public-safe discipline:
- keep the target seam visible before reading the final repair note
- keep owner-boundary preservation readable without private ops context
- keep ambiguity reduction separate from final quality proof
- keep the public repair read weaker than final owner proof

Replacement discipline:
- another repo may swap in local repair cases only if the same seam-clarity, scope-control, and trail-reviewability pressures remain visible
- do not replace this family with success-only cleanup stories that stop testing bounded repair itself
