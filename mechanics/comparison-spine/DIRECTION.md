# Comparison Spine Direction

Comparison Spine in `aoa-evals` should keep baseline, peer, and longitudinal
claims honest before any growth or regression reading is allowed.

Use this file for the package's current operating direction: read it after the
parent entry card and before `PARTS.md`, part contracts, source bundles,
decision records, and `PROVENANCE.md`.

## Source-of-truth split

- `README.md`: package entry card and shortest comparison route.
- `DIRECTION.md`: current operating direction.
- `PARTS.md`: active comparison part map.
- `parts/`: spine overview, fixed baseline, peer compare, and longitudinal
  window support.
- `PROVENANCE.md`: controlled bridge from active route to old comparison placement.
- `legacy/`: archive-local route for old comparison placement after
  `PROVENANCE.md`.
- `evals/`: source proof objects that state comparison claims.

## Current contour

- Keep comparison posture explicit before interpreting movement.
- Keep generated comparison spine below bundle source and report contracts.
- Keep fixed-baseline, peer-compare, and repeated-window support separate.
- Keep artifact-vs-process distinctions visible in reports and guides.

## Growth rule

Add comparison support only when a repeated comparison shape needs its own
fixture, report, generated readout, or validation. A single polished run does
not justify a growth claim or a new part.

## Stop-lines

| Pressure | Route |
| --- | --- |
| general improvement from one run | source bundle claim plus declared comparison mode |
| generated comparison reader as proof authority | source bundle, builder, and `build_catalog.py --check` route |
| baseline, peer, and longitudinal questions collapsed into one score | mode-specific part route plus bundle-local review |
| broad growth, repo-global score, runtime health, or sibling acceptance | comparison readout as evidence plus the owning growth, runtime, sibling, or release route |
