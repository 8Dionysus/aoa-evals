# Comparison Spine Proof Flow v1

This dossier defines the first shared read order across the current public comparison spine:
- `aoa-regression-same-task`
- `aoa-output-vs-process-gap`
- `aoa-longitudinal-growth-snapshot`
- `aoa-eval-integrity-check`

## Shared ladder

Read the current comparison spine in this order:
1. choose the one-run anchor surface first
2. choose `aoa-regression-same-task` for frozen-baseline comparison
3. choose `aoa-output-vs-process-gap` for side-by-side peer comparison
4. choose `aoa-longitudinal-growth-snapshot` for repeated-window movement
5. add `aoa-eval-integrity-check` whenever public wording, routing, or maturity posture could otherwise overstate the comparison read

## Distinctness boundary

The current spine is intentionally split:
- `aoa-regression-same-task` asks whether one candidate materially regressed against one frozen baseline on the same bounded task family
- `aoa-output-vs-process-gap` asks whether artifact-side and process-side readings diverge or align under matched conditions
- `aoa-longitudinal-growth-snapshot` asks whether ordered named windows show modest movement on one bounded workflow surface
- `aoa-eval-integrity-check` asks whether the public comparison wording and support artifacts still support only those bounded claims

## Anti-overread rule

Do not collapse this shared comparison flow into:
- one scalar score
- proof of broad capability growth
- automatic baseline status for draft bridge or longitudinal surfaces
- proof that a clean integrity read replaces the underlying comparison evidence
