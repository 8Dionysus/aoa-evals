# Distillation / Part Index

`mechanics/distillation/parts/` contains the active parts of the eval-side
Distillation proof operation.

The mechanic owns the route:

`distillation pressure -> selected support surface -> bundle-local review -> bounded abstraction/adoption read or owner handoff`

## Parts

| Part | Role | Active surfaces |
| --- | --- | --- |
| `compost-provenance` | Maintains provenance-preserving compost proof support for witness-derived note, synthesis, principle, or canon-facing candidate routes. | `mechanics/distillation/parts/compost-provenance/README.md` |
| `runtime-candidate-adoption` | Maintains reviewed runtime distillation candidate adoption proof support without upgrading candidate posture into settled truth. | `mechanics/distillation/parts/runtime-candidate-adoption/README.md` |

## Part Contract

Inputs are witness-facing sources, downstream compost artifacts, visible source
refs, review state, promotion boundary, runtime distillation candidate mappings,
reviewed memo adoption evidence, live receipt visibility, object-facing recall
surfaces, fixture families, and bundle-local report contracts.

Outputs are bounded Distillation proof readings, part-local fixture family
contracts, generated proof-artifact paths, report expectations, and owner
handoffs.

The current source proof bundles are
`aoa-compost-provenance-preservation` and
`aoa-memo-reviewed-candidate-adoption-integrity`.

Owner split stays explicit: `Agents-of-Abyss` owns center Distillation law;
Tree-of-Sophia owns compost and canon meaning; `aoa-memo` owns memory object
truth and durable review; `aoa-agents` owns runtime distillation artifact
contracts; `abyss-stack` owns runtime plumbing; KAG and owner repositories keep
their local truth; `aoa-evals` owns bounded proof interpretation and local
validation.

Stop-lines route stronger-owner pressure through the boundary routes below.

| Pressure | Owner route |
| --- | --- |
| summary-as-proof or raw deletion authority | source trace, source bundle, owner source-retention route |
| memory canon or live memory-ledger behavior | `aoa-memo` memory route |
| runtime activation or hidden runtime-store truth | `abyss-stack` runtime route |
| owner acceptance, owner-local adoption, or final promotion | owner repository acceptance route |
| ToS canon or compost authority | Tree-of-Sophia canon route |
| KAG bridge promotion or graph lift | `aoa-kag` graph-lift route |
| generic adoption readiness | `mechanics/experience/parts/adoption-federation/` route |
| memo recall after active recurrence routing | `mechanics/recurrence/parts/memory-recall/` route |
| nearby contradiction or base writeback proof | owning eval bundle or mechanic part route |

Validation routes through [AGENTS](AGENTS.md#validation), including the two
bundle-local eval checks, generated catalog check, and repo validation lane.

## Deferred Part Families

Memo contradiction, confirmed writeback-act integrity, witness trace integrity,
artifact/process pairing, and generic Experience adoption
remain outside this mechanic until local evidence proves a narrower
Distillation operation with source surfaces, inputs, outputs, owner split,
stop-lines, and validation.
