Start with this README for role and owned operation. Then read [DIRECTION.md](DIRECTION.md) for current operating direction, [PARTS.md](PARTS.md) for active parts, and [PROVENANCE.md](PROVENANCE.md) as the active-to-archive bridge for legacy or former-placement lookup.

## Role

`mechanics/distillation/` routes bounded proof work for
provenance-preserving abstraction and reviewed runtime-candidate adoption.

It receives distillation pressure when source traces, candidate posture,
receipt visibility, recall inspectability, and promotion boundaries need a
reviewable eval-side proof route.

## Owned Operation

`mechanics/distillation/` owns the eval-side Distillation proof operation:

`distillation pressure -> provenance-preserving or candidate-preserving proof question -> part-local support surface -> bundle-local review -> bounded abstraction/adoption read or owner handoff`

This package is AoA-aligned. It keeps the parent name `distillation` because
the local proof work materializes the center Distillation mechanic on the proof
side.

## Inputs

- witness-facing input, note, synthesis, principle candidate, or canon-facing
  candidate with visible source refs, review state, and promotion boundary;
- runtime `distillation_claim_candidate`,
  `distillation_pattern_candidate`, or `distillation_bridge_candidate`
  mappings with reviewed adoption, receipt visibility, and object-facing recall
  evidence;
- bundle-local fixture, runner, report, and generated-reader support that keeps
  the proof question reviewable.

## Outputs

- bounded provenance-preservation and reviewed-candidate-adoption proof
  readings;
- part-local fixture family contracts for distillation support surfaces;
- bundle-local report expectations and generated proof-artifact paths;
- owner handoff notes when ToS, aoa-memo, aoa-agents, abyss-stack, KAG, or
  owner-local adoption truth is stronger than the eval claim;
- owner handoff routes for raw-summary, memory-canon, runtime-activation,
  owner-acceptance, ToS-canon, KAG-lift, or generic-adoption-readiness pressure.

`aoa-evals` owns the bounded proof reading: whether provenance, candidate
posture, receipt visibility, recall inspectability, and promotion boundaries
stayed visible enough for review under the source bundle. Authority beyond
that proof reading routes through the stronger owner split above.

## Stop-Lines

Boundary routes keep Distillation pressure with the owner that can act on it:

| Pressure | Owner route |
| --- | --- |
| summary-as-proof pressure | source trace, source bundle, and bundle-local review route |
| raw deletion authority pressure | owner repository source-retention route |
| proof verdict without bundle-local review pressure | bundle-local review route |
| memory canon, recall sovereignty, or live memory-ledger pressure | `aoa-memo` memory route |
| runtime activation or hidden runtime-store pressure | `abyss-stack` runtime route |
| owner acceptance, owner-local adoption, or final promotion pressure | owner repository acceptance route |
| Tree-of-Sophia canon or compost authority pressure | Tree-of-Sophia canon route |
| KAG bridge promotion or graph lift pressure | `aoa-kag` graph-lift route |
| memo contradiction or confirmed writeback-act pressure | owning eval bundle or mechanic part route |
| memo recall pressure | `mechanics/recurrence/parts/memory-recall/` route |
