## Role

`mechanics/distillation/` routes bounded proof work for
provenance-preserving abstraction and reviewed runtime-candidate adoption.

It receives distillation pressure when source traces, candidate posture,
receipt visibility, recall inspectability, and promotion boundaries need a
reviewable eval-side proof route.

## Owned Operation

`mechanics/distillation/` owns the eval-side Distillation proof operation:

`distillation pressure -> provenance-preserving or candidate-preserving proof question -> part-local support surface -> bundle-local review -> bounded abstraction/adoption read or owner handoff`

This package is AoA-aligned. It keeps the parent name `distillation` because
the local proof work materializes the center Distillation mechanic on the proof
side.

## Inputs

- witness-facing input, note, synthesis, principle candidate, or canon-facing
  candidate with visible source refs, review state, and promotion boundary;
- runtime `distillation_claim_candidate`,
  `distillation_pattern_candidate`, or `distillation_bridge_candidate`
  mappings with reviewed adoption, receipt visibility, and object-facing recall
  evidence;
- bundle-local fixture, runner, report, and generated-reader support that keeps
  the proof question reviewable.

## Outputs

- bounded provenance-preservation and reviewed-candidate-adoption proof
  readings;
- part-local fixture family contracts for distillation support surfaces;
- bundle-local report expectations and generated proof-artifact paths;
- owner handoff notes when ToS, aoa-memo, aoa-agents, abyss-stack, KAG, or
  owner-local adoption truth is stronger than the eval claim;
- no proof verdict from raw summary alone, memory canon, runtime activation,
  owner acceptance, ToS canon, KAG lift, or generic adoption readiness.

## Stop-Lines

This package supports bounded eval-side proof only. Keep these claims outside
this package:

- summary-as-proof or summarization as distillation proof;
- raw deletion authority;
- proof verdict without bundle-local review;
- memory canon, recall sovereignty, or generic live memory-ledger behavior;
- runtime activation or hidden runtime-store truth;
- owner acceptance, owner-local adoption, or final promotion;
- Tree-of-Sophia canon or compost authority;
- KAG bridge promotion or graph lift;
- that memo contradiction or confirmed writeback-act integrity belongs here
  merely because the word memo appears nearby;
- that memo recall belongs here after it has an active
  `mechanics/recurrence/parts/memory-recall/` route.
