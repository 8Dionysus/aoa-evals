# memo-reviewed-candidate-adoption-guardrail-v1

Shared fixture family for `aoa-memo-reviewed-candidate-adoption-integrity`.

Use this family when the bounded question is whether reviewed distillation
claim, pattern, and bridge candidates can stay inspectable from runtime mapping
through reviewed memo adoption, live receipt visibility, and object-facing
recall without pretending that promotion is already solved.

Canonical case archetypes:

- explicit reviewed-candidate runtime mapping for claim, pattern, and bridge
- reviewed anchor that keeps adoption subordinate to the route boundary
- adopted candidate object with visible provenance and lifecycle posture
- live receipt publication step that keeps the adopted path inspectable later
- closed-circle read that returns from candidate mapping into object-facing
  recall without widening into promotion or ledger claims

Family invariants:

- reviewed-candidate adoption stays weaker than final promotion
- runtime mapping, reviewed anchor, adopted object, receipt visibility, and
  recall posture remain inspectable on the same bounded path
- adopted candidate surfaces remain provenance-visible and lifecycle-explicit
- receipt visibility remains reviewable when adoption is claimed
- the path stays weaker than confirmed memo-surviving-event writeback,
  contradiction resolution, or graph-lift completion

Replacement boundary:

- local repos may replace the concrete cases only if they preserve the same
  reviewed-candidate adoption question and still exercise all five bounded
  pressures above
- replacements must stay public-safe and must not depend on hidden runtime
  stores, private receipts, or unreviewed promotion shortcuts
