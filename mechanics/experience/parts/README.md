# Experience / Parts Route

`mechanics/experience/parts/` is the lower index for active eval-side
Experience proof parts. Use it after the parent Experience route has selected a
protocol, certification, adoption, governance, or office proof operation and
the next agent needs the exact part, payload home, owner route, tool lane, and
validation lane.

## Operating Card

| Field | Route |
| --- | --- |
| role | lower index for Experience verdict support and boundary proof parts |
| input | verdict packet, certification gate packet, adoption request, consent record, shadow proof, compatibility ref, governance verdict, runtime-boundary ref, office contour, release train packet, rollback drill, watchtower alarm, or owner-source ref |
| output | part README route, schema/example contract, fixture family route, bounded proof reading, generated catalog route, owner handoff, or bundle-local review route |

| tools | part-local tests, generated catalog checks, repo validator, and semantic AGENTS validator through `mechanics/experience/AGENTS.md#validation` |
| validation | `mechanics/experience/parts/AGENTS.md#validation` and `mechanics/experience/AGENTS.md#validation` |
