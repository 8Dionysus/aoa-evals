# Replay History Integrity Verdict

## Role

This note scopes replay-history integrity verdicts for governance/runtime proof.

Replay verdicts verify `prev_hash` chains, event hashes, and replay packet
continuity as bounded evidence.

## Reads

Use this surface when a governance or runtime-boundary packet cites replay
history and an agent needs to know what `aoa-evals` can inspect.

## Boundary

`aoa-evals` may check replay evidence shape and integrity signals. Runtime
owners keep replay execution, event acceptance, and live state authority.

Runtime law remains reviewable. Owner meaning remains owner-local.

## Validation

Use the governance-runtime-boundary README. A replay integrity verdict supports
review; it does not accept or rewrite runtime history by itself.
