# Candidate Lineage V1

This shared family anchors `aoa-candidate-lineage-integrity`.

Use it when the public question is:
- did one bounded growth-refinery lineage stay structurally coherent across `cluster_ref`, reviewed `candidate_ref`, staged `seed_ref`, and downstream owner evidence without laundering gaps into false continuity?

Canonical case archetypes:
- one clean forward chain where each stage appears in order
- one dropped branch that stays explicitly dropped instead of silently disappearing
- one superseded branch that keeps the replacement trail visible
- one missing-stage failure where downstream identity appears before its prerequisite stage

Public-safe discipline:
- keep stage order readable without private owner receipts
- keep terminal branch markers visible enough for an outside reviewer
- keep owner-shaping context visible without treating it as owner proof
- keep the public integrity read weaker than owner-fit or final object proof

Replacement discipline:
- another repo may swap in local lineage cases only if the same stage-order, gate, and terminal-branch pressures remain visible
- do not replace this family with owner-quality or object-quality cases that stop testing lineage coherence itself
