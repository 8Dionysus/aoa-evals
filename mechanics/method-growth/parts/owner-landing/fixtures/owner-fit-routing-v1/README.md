# Owner Fit Routing V1

This shared family anchors `aoa-owner-fit-routing-quality`.

Use it when the public question is:
- did a reviewed growth-refinery candidate land on one bounded owner hypothesis with one plausible nearest wrong target and enough evidence to keep derivative repos derivative?

Canonical case archetypes:
- one clearly correct owner target with specific bounded evidence
- one ambiguous case that still resolves honestly with a real nearest wrong target
- one derivative-repo temptation where `aoa-routing` or `aoa-kag` looks plausible but must remain derivative
- one thin-evidence case where the routing note sounds neat but stays weaker than the evidence

Public-safe discipline:
- keep the chosen owner and nearest wrong target readable on the same case surface
- keep derivative-repo exclusions visible without private governance context
- keep owner-shape reasoning bounded rather than inflated into final quality proof
- keep the public routing read weaker than final owner-object proof

Replacement discipline:
- another repo may swap in local owner-fit cases only if the same owner-choice, nearest-wrong, and derivative-boundary pressures remain visible
- do not replace this family with generic usefulness or output-quality cases that stop testing owner-fit routing itself
