# local-text-contract-v1

This shared fixture family defines a compact local text contract surface.

It is intentionally small and public-safe.
The goal is not to test general intelligence.
The goal is to keep four brittle contract pressures reviewable on one compact
surface:

- exact literal reply
- bounded repo routing
- bounded repo choice
- constrained JSON decision

## Required pressures

Any local replacement must preserve all four pressures.

### 1. Exact literal reply

The case must require one exact literal string with no wrapper text, apology,
or explanation.

### 2. Bounded repo routing

The case must require one route chosen only from a small explicitly named repo
set.

### 3. Bounded repo choice

The case must require one answer chosen only from a small allowed option set.

### 4. Constrained JSON decision

The case must require compact valid JSON with explicit keys and bounded values.

## Public-safe rule

The family must stay reviewable without private prompts, hidden repo knowledge,
or host-local secrets.

## Bounded interpretation rule

This family may support:

- compact contract-fit review
- cautious side-by-side reading of two local text lanes

This family may not support:

- broad model ranking
- universal latency claims
- rich-answer quality claims
