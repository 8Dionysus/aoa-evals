# Return Anchor V1

This shared family anchors `aoa-return-anchor-integrity`.

Use it when the public question is:
- did the route name a real return reason, point to a real anchor, rebuild only bounded context, and then re-enter honestly or stop honestly?

Canonical case archetypes:
- one return triggered by lost verification or phase posture that re-enters through a valid bounded plan anchor
- one return triggered by source or boundary confusion that must rebuild from explicit anchor surfaces rather than hidden continuity
- one decorative-anchor case where the re-entry looks smooth but the visible rebuild evidence is thinner than the prose implies
- one no-trustworthy-anchor case where safe stop is the honest outcome and repeated return pressure must stay visible rather than buried

Public-safe discipline:
- keep the return reason explicit and reviewable
- keep anchor refs inspectable without private runtime continuity
- keep the bounded rebuild surfaces visible before reading the final re-entry note
- keep safe-stop and repeated-return posture legible enough that smooth prose cannot hide loop theater

Replacement discipline:
- another repo may swap in local return-aware cases only if the same four anchor-integrity pressures remain visible
- do not replace this family with checkpoint-only restart cases, approval-only cases, or final-answer-quality cases that no longer test return-anchor integrity
