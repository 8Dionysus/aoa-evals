# Titan Direction

Titan in `aoa-evals` should keep Titan boundary seed canaries shaped and
bounded while making clear that canaries are a part, not the parent mechanic
and not incarnation proof.

Use this file for the package's current operating direction: read it after the
parent entry card and before `PARTS.md`, part contracts, source bundles,
decision records, and `PROVENANCE.md`.

## Source-of-truth split

- `README.md`: package entry card and shortest Titan proof-seed route.
- `DIRECTION.md`: current operating direction.
- `PARTS.md`: active Titan part map.
- `parts/`: seed-boundary support and seed YAML route.
- `PROVENANCE.md`: controlled bridge from active route to old `evals/` placement and historical
  `titan-canaries` parent wording.
- `legacy/`: archive-local route for historical `titan-canaries` parent
  wording after `PROVENANCE.md`.
- `aoa-agents`: stronger Titan role classes, bearer identity, summon boundary law, and incarnation posture.

## Current contour

- Keep the parent name `titan`; canaries and seeds live under
  `parts/seed-boundary/`.
- Keep seed YAML shape validated and weaker than executable scorer suites.
- Keep future executable scoring as future route work, not current proof.
- Keep Titan identity, summon runtime, memory authority, and runtime roster
  truth with stronger owners.

## Growth rule

Add Titan parts only when seed-boundary pressure becomes a separate repeated
proof operation with inputs, outputs, owner split, stop-lines, and validation.
`titan-canaries` parent pressure routes through `PROVENANCE.md`, legacy indexes,
and validator-backed replacement paths.

## Stop-lines

| Pressure | Route |
| --- | --- |
| incarnation completion or broad Titan proof | keep seed canaries not incarnation proof; route stronger claims to `aoa-agents` |
| summon runtime authority, hidden arena safety, or runtime roster truth | route to runtime and summon owners before proof wording changes |
| memory authority | route to `aoa-memo` before adoption or proof strengthening |
