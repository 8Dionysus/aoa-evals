# Titan Parts

`parts/` contains Titan proof-seed artifacts that are narrower than the parent
mechanic but too operational to stay at package root.

The current payload form is seed canaries. The parts district is not named after the canary artifact form:
canary YAML files belong under the
`seed-boundary` part until a future scorer, fixture, report, or validator
contract proves a different part.

Read [../PARTS.md](../PARTS.md) for the active map.
