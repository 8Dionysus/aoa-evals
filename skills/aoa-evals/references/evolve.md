# Evolve owner proof

## Entry gates

Require a stable bounded need, accepted review or recorded duplicate/no-fit,
resolved owner, acceptance criteria, explicit effect authority, and every
human confirmation required by root `AGENTS.md`. Stop `blocked` when any gate
is missing.

## Procedure

1. Run the declared `source` command from `references/source-return.md` once.
   Read the returned owner `AGENTS.md` before judging authority or required
   human confirmation. Do not use catalog search unless comparison with source
   bundles is material to the requested evolution. Treat a `session:` candidate
   ID as an owner-routed eval object, not as a request to load session-memory;
   use session-memory only when raw or segment evidence must actually be
   retrieved.
2. Check the entry gates against the exact supplied owner sources. When the
   request forbids changes, effect authority is absent, or accepted owner
   review and required human confirmation are missing, return one bounded
   blocker after reading only the sources needed to establish those gaps. Do
   not inventory alternative bundles, run the motivating-case suite, execute
   readiness or validation commands, or draft a source change after a failed
   gate.
3. Only after every entry gate passes, decide the owner before the artifact:
   - local pressure, suites, fixtures, and reports stay in the origin repo port;
   - candidate packets and Eval Forge worksheets remain non-proof;
   - reusable central proof meaning belongs in a source eval bundle;
   - repeatable proof operations belong to their mechanic;
   - generated readers, receipts, MCP, and KAG remain derived or access layers.
4. Compare reuse, extension, a new bundle, maturity change, supersession,
   retirement, and no change. Prefer the narrowest existing owner surface that
   already answers the bounded need.
5. Manually exercise the motivating case, strongest negative, nearest competing
   bundle, evidence-gap case, and intended consumer.
6. Add permanent automation only when those cases expose a stable, long-lived
   invariant and the current task authorizes the write. Do not create a
   validator to make an unresolved trial look green.
7. When authorized, change owner source before generated companions. Rebuild
   only declared projections, inspect the actual source and outputs, and state
   what green checks do not prove.
8. Otherwise return an owner-ready bounded proposal, `no_change`, handoff, or
   blocker. Do not silently broaden the effect.

## Verification and stop

Report the exact source change, actual effects, manual cases, generated
companions rebuilt, skipped checks, claim limit, rollback posture, and next
owner. Stop after one bounded owner decision or source-first change.
